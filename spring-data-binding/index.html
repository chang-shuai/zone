SpringMVC中的请求参数绑定

[TOC]

一、什么是请求参数绑定

  • 请求参数格式

    默认是 key=value 格式，比如：http://www.chang.com:8080?id=1&pricee=1200

  • 请求参数值的数据类型：都是字符串类型的各种值

  • 请求参数值要绑定的目标类型

    Controller 类中的方法参数，比如简单类型、POJO类型、集合类型等。

  • SpringMVC 内置的参数解析组件

    默认内置了24种参数解析组件（ArgumentResolver）

  • 什么是参数绑定？

    就是将请求参数串中的 value 值获取到之后，再进行类型转换，然后将转换后的值赋值给Controlle r类中方法的形参，这个过程就是参数绑定。

二、HTTP请求类型的影响

  • @RequestMapping

    所有请求类型都可以处理, 无论是 GET 请求的url传递参数, 还是 POST 的请求体传递参数, 都会被解析绑定.

    因为用 HttpServletRequest#getParameter() 可以获取任何方式传递过来的参数.

  • @GetMapping

    这个注解只处理 GET 请求, 但是如果 GET 请求的请求体中有数据, 也可以绑定成功,原理同 @RequestMapping

  • @PostMapping

    这个注解只处理 POST 请求, 但是如果 POST 请求的 url 中有数据, 也可以绑定成功, 原理同 @RequestMapping

2. 简单数据类型

简单类型指的就是 8种基本类型 数据以及它们的包装类，还有 String 类型。

在 SpringMVC 中，对于 Java 简单类型的参数，推荐的参数绑定方式有两种：

  • 直接绑定
  • 注解绑定

直接绑定

http 请求参数的 key 和 Controller 方法的 形参名 一致,

例如：http://localhost:8080/param/getItem?id=1 ，这里的 key 就是 id。

Controller 的形参为 Integer id , 它和请求参数的 key 一致，所以直接绑定成功。示例如下：

注解绑定

请求参数的 key 和 Controller 方法的形参名称不一致时，需要使用 @RequestParam 注解将 key 转换一下

例如: http://localhost:8080/param/getItem?itemId=1

注：RequestParam注解介绍：

  • value：参数名字，即入参的请求参数名字，如 value=“itemId”表示请求的参数中的名字为itemId 的参数的值将传入；
  • required：是否必须，默认是 true，表示请求中一定要有相应的参数，否则将报：HTTP Status 400 - Required Integer parameter ‘XXXX’ is not present；
  • defaultValue：默认值，表示如果请求中没有同名参数时的默认值。

绑定 POJO 类型

  • 控制器方法的参数类型是 POJO 类型。
  • 要求表单中 参数名 和POJO 类的 属性名 保持一致。

例如: http://localhost:8080/param/bindPojo?id=3&name=小米&price=3760

如果 Controller 中方法的形参既有 基本类型 也有 POJO类型 , 那么2个形参都会进行绑定

例子中的 id 和 Item中 属性id 都会被赋值.

如果 Controller 中方法的形参有多个 POJO , 那么2个 POJO 都会进行绑定

四、自定义数据类型转换与绑定 (重点)

4种方式

  1. PropertyEditor
  2. Converter (Spring3.1+)
  3. HttpMessageConverter
  4. Formatter

全部用于数据类型的转换

  • PropertyEditor 是JDK自带的, 其它的都是Spring提供接口.

  • PropertyEditor, Formatter 用于String和所需类型之间的转换;

  • Converter 可以是任意类型间的转换,包括了String与其它类型的转换;

  • HttpMessageConverter 用于把 HttpMessage (包括 HttpRequest 和 HttpResponse )中的内容与所需类型之间的转换.

它们都可以由Spring自动调用(在需要类型转换的时候), Formatter 还可提供注解形式,用注解指明调用.

调用顺序

遇到类型转换时, 它们的调用顺序如下(后者只有在前者未找到时执行):

  1. 使用 @InitBinder 注册的 PropertyEditor (只在局部注册,优先使用)

  2. Formatter 加入了 Locale 信息, 所以会优先使用, 如果使用了注解形式, 优先使用注解

  3. Converter

  4. 全局的 PropertyEditor (注意Spring未使用 PropertyEditorManager , 只支持在Model的同包下查找ModelEditor)

注意: HttpMessageConverter主要用于 HttpMessage 跟对象的转换,只有在使用 @RequestBody, HttpEntity<T>, @ResponseBody 实会用到。
